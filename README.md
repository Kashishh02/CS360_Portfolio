App Summary and User Needs
This app is a simple inventory management system. It allows users to log in, view items, and manage quantities. The goal was to make inventory tracking easy and quick to use without confusion. The app focuses on simplicity so users can understand it immediately.

Screens and Features
The app includes a login screen, a dashboard screen, and an SMS permission screen. The login screen allows access, the dashboard shows inventory in a table format with options to add or delete items, and the SMS screen handles notification permissions.
The design focuses on clear layout, readable text, and easy-to-find actions. Buttons and features are placed where users expect them, which makes the app easier to use.

Development Approach
I built the app step by step, focusing on one feature at a time. I made sure each screen worked before moving to the next. This helped reduce errors and made debugging easier. I also focused on connecting activities and handling button actions properly.

Testing Process
I tested the app using the emulator and checked each feature individually. I made sure navigation worked and buttons responded correctly. Testing helped identify issues early and ensured the app worked as expected.

Challenges and Innovation
One challenge was organizing the UI so it looked clean and not crowded. I had to adjust spacing and layout multiple times. I also worked through issues with navigation between screens. I solved these by testing often and making small improvements.

Strengths and Success
The strongest part of my app is the UI design. The layout is simple, clear, and easy to use. The dashboard especially shows information in a way that is easy to understand, which reflects good user-centered design.
